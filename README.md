# Personal Budget Maker

A full-stack budgeting application built on the **zero-based budget** principle —
every dollar of income is assigned a job before the month begins, so the balance
always reaches exactly **$0**.

**Stack:** Python · FastAPI · SQLAlchemy 2 (async) · React · TypeScript · Vite

---

## What is a zero-based budget?

A budget is a written, intentional plan for your money — made *before* the month
begins, not after. The zero-based approach assigns every dollar of income to a
category (expenses, savings, or goals) until nothing is left unaccounted for:

```
Income − (Expenses + Savings + Goals) = $0
```

| Without a budget | With a budget |
|---|---|
| You wonder where your money went | Every dollar was told where to go |
| Unexpected expenses feel like emergencies | A buffer category absorbs surprises |
| Saving feels impossible | Savings is just another line item |
| Goals stay vague ("someday…") | Goals have a monthly dollar amount |
| Debt tends to grow | Spending stays within actual income |

**Surplus (balance > $0):** unassigned money — allocate it intentionally (vacation
fund, retirement, extra debt payment).  
**Deficit (balance < $0):** your plan spends more than you earn — find a category
to trim or a way to grow income.

---

## Features

- **Income sources** — wages, dividends, side hustles, miscellaneous; each with a
  pay frequency (daily / weekly / biweekly / monthly) so all amounts are normalised
  to the same period for comparison
- **Expense catalogue** — 9 categories, 54 default line items, all customisable
  per user via a copy-on-write pattern (defaults are never mutated; reset is instant)
- **Period budgets** — create a budget for any time window; record planned amounts
  per income source and per expense item; mark items fixed or variable
- **Budget report** — total income, total expenses, balance, and per-category
  breakdowns; every figure normalised to the budget period
- **JWT authentication** — register, login, access-token + refresh-token flow
- **Interactive API docs** — Swagger UI and ReDoc auto-generated by FastAPI

## API endpoints

All protected endpoints require `Authorization: Bearer <access_token>`.

### Authentication — `/auth`

| Method | Path | Auth | Description |
|---|---|---|---|
| `POST` | `/auth/register` | — | Create a new account |
| `POST` | `/auth/login` | — | Obtain access + refresh tokens |
| `GET` | `/auth/me` | ✓ | Return the current user's profile |

### Income sources — `/income-sources`

| Method | Path | Description |
|---|---|---|
| `GET` | `/income-sources` | List all income sources for the current user |
| `POST` | `/income-sources` | Create an income source |
| `GET` | `/income-sources/{id}` | Get a single income source |
| `PUT` | `/income-sources/{id}` | Update an income source |
| `DELETE` | `/income-sources/{id}` | Soft-delete an income source |

**Supported frequencies:** `DAILY` · `WEEKLY` · `BIWEEKLY` · `MONTHLY`  
**Income types:** `WAGES` · `INTEREST_DIVIDEND` · `SIDE_HUSTLE` · `MISCELLANEOUS`

### Expense catalogue — `/categories` · `/items`

| Method | Path | Description |
|---|---|---|
| `GET` | `/categories` | List all categories with their items (defaults + user overrides merged) |
| `POST` | `/categories` | Create a custom category |
| `PUT` | `/categories/{id}` | Rename a category |
| `DELETE` | `/categories/{id}` | Remove a custom category |
| `GET` | `/categories/{id}/items` | List items in a category |
| `POST` | `/categories/{id}/items` | Add a custom item to a category |
| `PUT` | `/items/{id}` | Rename / hide / show an item (copy-on-write for defaults) |
| `DELETE` | `/items/{id}` | Remove a custom item |
| `POST` | `/items/reset-defaults` | Wipe all user overrides — restore factory defaults |

### Budgets — `/budgets`

| Method | Path | Description |
|---|---|---|
| `GET` | `/budgets` | List all budgets for the current user |
| `POST` | `/budgets` | Create a new budget for a period |
| `GET` | `/budgets/{id}` | Get a budget |
| `PUT` | `/budgets/{id}` | Update a budget |
| `DELETE` | `/budgets/{id}` | Delete a budget |
| `GET` | `/budgets/{id}/income` | List income entries for the budget |
| `PUT` | `/budgets/{id}/income/{source_id}` | Set planned amount for an income source |
| `DELETE` | `/budgets/{id}/income/{source_id}` | Remove an income entry |
| `GET` | `/budgets/{id}/expenses` | List expense entries for the budget |
| `PUT` | `/budgets/{id}/expenses/{item_id}` | Set planned amount for an expense item |
| `DELETE` | `/budgets/{id}/expenses/{item_id}` | Remove an expense entry |
| `GET` | `/budgets/{id}/report` | Full budget report: income, expenses, balance |

### Health

| Method | Path | Description |
|---|---|---|

## Environment variables

### Backend

| Variable | Required | Description |
|---|---|---|
| `DATABASE_URL` | Yes | Full SQLAlchemy-compatible connection string. Use `sqlite+aiosqlite:///./budget.db` for local dev. Render injects the PostgreSQL string automatically. |
| `SECRET_KEY` | Yes | Random secret used to sign JWTs. Generate with `openssl rand -hex 32`. |
| `ACCESS_TOKEN_EXPIRE_MINUTES` | No | JWT lifetime in minutes (default: 30). |

The `config.py` validator automatically rewrites `postgres://` to
`postgresql+asyncpg://`, so Render's connection string works without manual
editing.

### Frontend

| Variable | Required | Description |
|---|---|---|
| `VITE_API_BASE` | No | Base URL of the API. Omit in local dev — the Vite proxy handles it. Set to the Render URL for production builds. |

## Getting started

### Option A — Local dev with SQLite (simplest)

```bash
# 1. Clone and install dependencies
git clone https://github.com/laislodi/personal-budget-maker.git
cd personal-budget-maker
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# 2. Create a .env file
echo 'DATABASE_URL=sqlite+aiosqlite:///./budget.db' > .env
echo 'SECRET_KEY='$(openssl rand -hex 32) >> .env

# 3. Seed default categories and expense items
python -m seed.defaults

# 4. Start the API
uvicorn app.main:app --reload
# → http://localhost:8000
# → http://localhost:8000/docs  (Swagger UI)
```

### Option B — Local dev with PostgreSQL (closer to production)

```bash
# 1. Copy the example env file and fill in your values
cp .env.example .env          # create this file — see Environment variables above

# 2. Start the database
docker compose up -d

# 3. Run migrations and seed
alembic upgrade head
python -m seed.defaults

# 4. Start the API
uvicorn app.main:app --reload
```

`.env` for Docker Compose:
```
POSTGRES_USER=budget_user
POSTGRES_PASSWORD=changeme
POSTGRES_DB=budget_db
DATABASE_URL=postgresql+asyncpg://budget:changeme@localhost:5432/budget_db
SECRET_KEY=<output of: openssl rand -hex 32>
```

### Frontend

```bash
npm install

# For local API:
npm run dev          # proxies /api/* → http://localhost:8000

# For Render API:
echo 'VITE_API_BASE=https://personal-budget-maker.onrender.com' > .env.local
npm run dev
```

## Deploying to Render

The repository includes a `render.yaml` Blueprint that provisions a web service
and a managed PostgreSQL database in one click.

1. Push this repository to GitHub.
2. Go to [Render](https://render.com) → **New → Blueprint**.
3. Connect the repository — Render reads `render.yaml` and creates both resources.
4. `SECRET_KEY` is auto-generated; `DATABASE_URL` is injected from the managed DB.
5. On every deploy, `seed/defaults.py` runs automatically before the server starts.

Live API (example): `https://personal-budget-maker.onrender.com/docs`
